<task type="auto">
  <name>Task 2: Create content script with speed injection logic</name>
  <files>content.js</files>
  <action>
Create `content.js` implementing the complete speed injection logic. Follow the research code example closely but ensure all details are correct:

1. **Constants at top:**
   - TARGET_SPEED = 1.5 (hardcoded per locked decision -- Phase 2 adds chrome.storage)
   - MAX_RETRIES_PER_SECOND = 10 (throttle limit per locked decision)
   - THROTTLE_WINDOW_MS = 1000

2. **WeakSet for tracking:** `const processedVideos = new WeakSet()` to avoid duplicate listeners on the same video element (handles Pitfall 4 from research).

3. **applySpeed(video) function:** Sets `video.playbackRate = TARGET_SPEED`. Simple, no frills.

4. **initSpeedControl(video) function:**
   - Guard: `if (processedVideos.has(video)) return` then `processedVideos.add(video)`
   - Call `applySpeed(video)` immediately
   - Add `loadedmetadata` listener (once) that calls `applySpeed(video)` (handles Pitfall 5 -- speed not sticking before metadata loads)
   - Attach ratechange enforcement listener (inline, not separate function):
     - Guard flag `settingSpeed` (boolean) to prevent infinite loop (Pitfall 1)
     - Skip if `video.playbackRate === TARGET_SPEED`
     - Throttle: reset `retryCount` if `Date.now() - windowStart >= THROTTLE_WINDOW_MS`
     - Bail if `retryCount >= MAX_RETRIES_PER_SECOND`
     - Increment retryCount, set guard true, set playbackRate, set guard false

5. **Initial scan:** `document.querySelectorAll('video').forEach(initSpeedControl)`

6. **MutationObserver:**
   - Observe `document.documentElement` with `{ childList: true, subtree: true }`
   - In callback: iterate addedNodes, check `node.nodeType === Node.ELEMENT_NODE`
   - If node IS a video tag: `initSpeedControl(node)`
   - If node contains children: `node.querySelectorAll('video').forEach(initSpeedControl)`

IMPORTANT constraints:
- Silent operation -- NO console.log, NO visual feedback, NO alerts (per locked decision)
- Speed range: 0.5x min, 4x max -- not enforced in Phase 1 (hardcoded 1.5x is within range) but the constants should be clean for Phase 2 to extend
- Do NOT use Vimeo Player SDK -- direct HTML5 video.playbackRate only (per locked decision)
  </action>
